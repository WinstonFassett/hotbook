layerchart-direct · Path A spike

Svelte 5 + bireactive · two web-component instances over a shared writable tree

Instance A — Treemap (web component)

Hover a tile and hold cmd/ctrl+wheel to scrub — the gesture target locks while the modifier is held, so re-layout (or shrinking to zero) can't break the gesture. Click to focus, then Tab/Shift+Tab to walk cells, arrow keys to nudge value (±1, shift ±5). Works on groups too — the Num.lens redistributes children proportionally and sibling totals re-derive automatically.

Instance C — Real LayerChart <Chart> + <Treemap>, same tree

Same shared bireactive tree, but the layout is run by the actual LayerChart components consumed from npm — not our local d3.treemap call. Proves the Path A bet: LayerChart's hierarchy vocabulary works when the data underneath is writable cells. Same gestures.

Sunburst — LayerChart <Partition> + <Arc>, same tree

Partition laid out into polar coords [0, 2π] × [0, R], rendered as nested arcs. Same shared cells: scrubbing a slice here updates every other instance.

Icicle — LayerChart <Partition> in cartesian, same tree

Same Partition layout as the sunburst, rendered as stacked bars (depth flows top-to-bottom, siblings span horizontally). Cartesian variant of the same data structure.

Circle Pack — LayerChart <Pack> + <Circle>, same tree

Hierarchy as nested circles via d3.pack. Different geometry, same gesture model — hover + cmd/ctrl+wheel scrubs the targeted node; siblings repack on every change.

Instance B — Same web component, same underlying tree

Both instances mount the same Svelte component, which imports the same module-scope tree from tree.ts. Editing a tile in one instance updates the other in real time — proving cross-instance reactivity via shared bireactive cells, with zero event plumbing between the components.
